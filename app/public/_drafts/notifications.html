Open questions (need user input)

1. SMTP — self-hosted Postfix vs relay?
Lean: Mailgun-style relay. Home IPs hit spam filters.
2. Failure behavior — silent vs logged?
Lean: logged. Surface in Reports → Data Quality.
3. Cron — node-cron in container vs QNAP scheduler?
Lean: node-cron in container. Self-contained.
4. Quiet hours — respect 10pm–7am?
Lean: yes. Notifications still created; only push delivery suppressed.

Locked agreements

  • Bell icon stays in nav header (already wired).
  • Page exists at /notifications.html.
  • Notifications fed by existing autoTodos triggers.
  • App-internal first. Email/SMS later.
  • One row per notification — group on read, not on create.
  • Manual mark-read (auto-read loses context).

Backend changes (anticipated)

Migration:

  • ALTER TABLE notifications ADD COLUMN delivery_status TEXT
  • ALTER TABLE notifications ADD COLUMN pushed_at DATETIME
  • ALTER TABLE notifications ADD COLUMN read_at DATETIME

New table possibly: notification_settings (per-trigger on/off, channels).

New endpoints:

  • GET /api/v1/notifications
  • GET /api/v1/notifications/count
  • POST /api/v1/notifications/:id/read
  • POST /api/v1/notifications/test

Out of scope

  • Push notifications (web/mobile push API)
  • SMS
  • Slack / Discord webhooks
  • Per-family-member routing
