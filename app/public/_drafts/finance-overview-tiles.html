#22 — Finance Overview tiles (locked)

Reference doc — captures the agreed Finance Overview tile design so future drops don't drift. Anything that deviates from this is a redesign requiring Al's explicit go-ahead, not an iteration.

The 6 tiles (locked order)

# Name Hero metric Pill Click target
T1 Net Worth Total $ (assets − liab + investments) MoM delta networth tab
T2 Cash Flow MTD Net signed (+/− $) vs last mo transactions tab
T3 Credit Cards Total owed (statement balance) Utilization % accounts tab
T4 Bank Accounts Total liquid (reserved — stale count) accounts tab
T5 Holdings Market value (reserved — YTD %) holdings tab
T6 HSA + LP-FSA Combined unreimbursed pool (reserved — FSA deadline) hsa tab

Per-tile spec

T1 — Net Worth

  • Hero: Total $ (assets − liabilities, includes investments)
  • Eyebrow: "Net worth · <generated_at date>"
  • Pill: MoM delta or "first month" if no prior snapshot
  • Dot: green if total ≥ 0, red if negative
  • Strip: "$X assets · $Y liabilities"
  • Click: → networth tab

T2 — Cash Flow MTD

  • Hero: Net signed (+$ / −$) for current MTD
  • Eyebrow: "Cash flow · MTD <Month>"
  • Pill: "vs last mo +/−$X" or "no data yet" if zero MTD activity
  • Dot: green if net ≥ 0, amber otherwise
  • Bar: in (credits) vs out (debits) horizontal split
  • Bar labels: "in $X" / "out $Y"
  • Strip: "<N> transactions this month"
  • Click: → transactions tab

T3 — Credit Cards

  • Hero: Total statement balance owed across active credit accounts
  • Eyebrow: "Credit cards · <N> active"
  • Pill: Utilization % — green <30, amber <60, red ≥60
  • Dot: red if next due ≤3d, amber if ≤7d, green otherwise
  • Strip: "<account> due in Nd · $M min" or "overdue Nd"
  • Click: → accounts tab
  • Earliest-due rule: min payment_due_date where balance > 0; tiebreak by larger min payment

T4 — Bank Accounts

  • Hero: Total liquid (Checking + Savings + Cash)
  • Eyebrow: "Banks · <N> account(s)"
  • Pill: (none in v.150 — reserved for stale-count later)
  • Dot: green
  • Rows: by_type breakdown — Checking $X, Savings $Y, Cash $Z
  • Click: → accounts tab

T5 — Holdings

  • Hero: Total market value across all positions
  • Eyebrow: "Holdings · <N> position(s)"
  • Pill: (none in v.150 — reserved for YTD % later)
  • Dot: green
  • Strip: "tap to manage" or "no holdings imported yet"
  • Click: → holdings tab

T6 — HSA + LP-FSA

  • Combined into one tile — locked
  • Hero: Combined unreimbursed pool
  • Eyebrow: "HSA + LP-FSA"
  • Pill: (none in v.150 — reserved for FSA deadline later)
  • Dot: green
  • Rows: HSA $X, LP-FSA $Y (only render rows for pots that exist)
  • Click: → hsa tab (becomes /medical.html#receipts when Path B ships)

What's NOT in the tile set (logged)

  • Subscriptions — already in main nav, duplicate signal
  • Budgets — Al said skip; add later if used
  • Gift cards — low signal as a tile
  • Trade terminal — its own page
  • Property — its own page

Visual rules (locked)

  • 5-zone template: eyebrow / hero / pill / strip / entities
  • Status dot in eyebrow, semantic colors (green / amber / red)
  • Status pill on right of hero, same color family as the dot
  • Hero number: Newsreader/Fraunces italic, 40px desktop / 36px mobile
  • Mini rows: DM Mono 12px; flex:1 label; right-aligned $ with min-width:54px
  • Strip line at bottom: mono 11px; red when alert
  • Mobile: 1-column grid below 700px viewport

Sample-data handling (post v.160/161)

  • No invented numbers anywhere. Ever.
  • Tile renders — placeholders until /finance/landing returns
  • If a tile has no real backing data, it stays at —
  • SAMPLE pill badge is OPTIONAL (currently in v.161) — Al has the call on whether to keep or remove. Default lean: remove.

Backend contract — GET /api/v1/finance/landing

{
  "generated_at":  "<ISO timestamp>",
  "net_worth":     { "total", "assets", "liabilities", "investment_total", "mom_delta" },

  "credit_cards":  { "count", "statement_balance", "utilization",
                     "soonest_due_date", "soonest_due_account", "min_payment" },
  "bank_accounts": { "count", "total", "by_type": [{ "type", "total" }] },
  "holdings":      { "positions", "market_value" },
  "hsa_lp_fsa":    { "hsa_count", "hsa_total", "lp_fsa_remaining", "combined" }
}
