#19 — Vehicles module

Status: spec ready · Priority: medium · Sources: chat 1 mockup

Per-vehicle profiles with extensible maintenance log, insurance, registration, attached documents. Mockup approved chat 1. Maintenance type list must be user-extensible (clarification from chat 1).

Entities

  • vehicles — make, model, year, VIN, plate, color, purchase date/price
  • vehicle_service — type, date, mileage, shop (FK contacts), amount, details, next_due_date, next_due_mileage
  • vehicle_service_types — seeded list, user can add new
  • vehicle_registration — state, expires
  • Insurance — reuses pattern from #20 insurance / historical tracking
  • Attachments use existing system, entity_type='vehicle'

  6. Build maintenance entry form: type dropdown with "+ add new", date, mileage, shop (contacts picker), amount, details, next-due fields. (Claude)
  7. Wire dashboard widget: vehicle items due soon. (Claude)
  8. Add Reports entry: vehicle service history per vehicle. (Claude)

Locked agreements

  • Vehicles is a top-level nav item, not buried in Settings.
  • Maintenance type list is editable (seed + add new pattern).
  • Shop name = contacts FK (contact_type='vendor' or new 'auto_shop').
  • Insurance reuses pattern from draft #20 — do not duplicate.
  • Registration expiry alert window: 60 days.
  • Maintenance due alert: 500 mi or 30 days, whichever first.
  • Documents use shared attachments (entity_type='vehicle').
