### Task 2: `emergency-info` + `family-snapshot` viewers (iterate all members)

Both share the same fetch pattern: Promise.all over all family members → per-member `/api/v1/family-snapshot/:id`.

**emergency-info spec:**

- Body: card-per-member layout — `<h3>` heading with display_name + relationship, then a small grid:
  - Date of birth (computed age in parens), gender

  - Active medications (count + first 3 names; "+N more" if longer)
  - Allergies (joined from medications array? actually from member.allergies — wait, on family_members there's no allergies field; pull from `medical/allergies` route OR use the snapshot's `conditions` filtered by category. SIMPLEST: pull from snapshot's `medications[].name` for med list. Allergies live on `kids` schema, not family_members — skip the Allergies row for non-kid members; show emergency_notes instead).
  - Conditions (count + first 3 names)
- Row click on a member's card → drilldown with full member detail (all fields from snapshot.member + full medications + full conditions + full visits)
- Empty state: `'No family members'` / `'Add family members in Settings → Family.'`

**family-snapshot spec:**
- No filters
- 4 cards: Total members · Total active medications (sum across all member.medications[]) · Total conditions tracked · Total visits this year (filter visits[].visit_date.startsWith(repCurrentYear()))
- Body: single table — Name / Relationship / Age (computed from DOB) / Active meds (count) / Conditions (count) / Visits this year (count) / Emergency notes (yes/no badge)
- Row click → drilldown with same member detail as emergency-info
- Empty state same shape as emergency-info

**Shared fetch helper** (define inside the viewer object — or pass via a small module-scope helper):
```js
async function loadAllMembers() {
  const fmRes = await fetch('/api/v1/data/table?name=family_members')
    .then(r => { if (!r.ok) throw new Error('HTTP ' + r.status); return r.json(); });
  const members = (fmRes.rows || []);
  const snapshots = await Promise.all(members.map(m =>
    fetch('/api/v1/family-snapshot/' + m.id)
      .then(r => r.ok ? r.json() : null)
      .catch(() => null)
  ));
  // Merge snapshot back onto member; skip nulls (member with snapshot fetch failure shows minimal data).
  return members.map((m, i) => ({ ...m, _snapshot: snapshots[i] || {} }));
}
```

Note: keep `loadAllMembers` SCOPED INSIDE the first viewer that defines it (no module-scope leak); the second viewer redefines it OR they share via the v.202 pattern of independent definition.
