## Endpoint reference (all pre-verified 200 on prod 2026-05-26)

| Slug | Endpoint | Returns |
|---|---|---|
| `hsa-spending` | `GET /api/v1/hsa/payments?year=<Y>` × 5 years parallel | Array per year. Client aggregates by year: count, you_paid sum, reimbursed sum. |
| `meds-active` | `GET /api/v1/medical/medications` | Array. Client filters `status === 'Active'`. Fields: id, patient, family_member_id, name, dosage, frequency, physician, start_date, end_date, status, next_refill_date, refills_remaining. |
| `visits-history` | `GET /api/v1/medical/notes` | Array enriched server-side with `family_member_name`, `attachment_count`, `linked_conditions[]`, `physician_name`, `practice_name`. Fields: id, visit_date, patient, family_member_name, physician_name, practice_name, follow_up_needed, attachment_count, linked_conditions. |
| `immunizations` | `GET /api/v1/medical/immunizations` | Array. Server orders by next_due_date ASC then date_given DESC. Fields: id, family_member_id, vaccine_name, date_given, dose_number, lot_number, location_text, next_due_date, notes. |
| `procedures` | `GET /api/v1/medical/procedures` | Array. Server orders planned first then by procedure_date DESC. Fields: id, family_member_id, procedure_name, procedure_date, facility_text, procedure_type, status, outcome_notes. |
| `receipts-missing` | `GET /api/v1/hsa/payments?year=<Y>` | Array. Client filters `receipt_saved !== 1`. Fields: id, date, patient, provider, category, you_paid, hsa_eligible, receipt_saved, reimbursed. |

For the family-member-aware viewers (`meds-active`, `immunizations`, `procedures`), the data has `family_member_id` but no display name. Use `/api/v1/data/table?name=family_members` parallel fetch to build an id→name map. That endpoint already exists.
